# 中文幼儿园绘本创作工作流（自主翻阅 · 政策驱动 · 多 Agent）

为**中文幼儿园儿童（3–6 岁）**创作面向**自主翻阅**的绘本——孩子大多不识字，靠**看图**读懂故事。本工程以教育部《3—6 岁儿童学习与发展指南》为政策基座，由一支**有人格的子 Agent 团队**分工编排创作，从源头规避「一个 AI 从头做到尾」的幻觉。

> **北极星：让孩子只看图，就能把故事讲对。**　画面承担叙事，文字极简，分小/中/大班三段差异化。

## 怎么用
在 Claude Code / Codex 里说「**做一本中文绘本**」或「给幼儿园孩子做一本能自己看图读懂的书」，即触发 Skill `chinese-picture-book`。它会以**主编**身份，按流程依次调度子 Agent，并在中途跑两道把关门，最后交付可出图、可排版成 PDF 的「成书包」。

## 团队（`.claude/agents/`）
- **主编·墨白** — 总制片：项目卡、终检与组装；路由与把关
- **育儿顾问·苗老师** — 政策/年龄段把关（政策门）
- **作家·林禾** — 故事圣经 + 分龄极简文字
- **画面叙事设计师·岛崎** — 画面脚本（画面即叙事，**脊梁**）
- **绘画师·青柚** — 每页配图 prompt + 一致性工程
- **校对·一字** — 分龄语言校对
- **童读小测者·豆豆** — 只看图走查（**理解门**：只看图能不能读懂）

## 流程（带两道门）
```
项目卡(锁年龄段) →[确认]→ 政策预简报 → 故事圣经 →[政策门A][确认]→
画面脚本 → 每页文字 →[校对B][政策门C][理解门D·核心环路]→ 配图Prompt → 终检·成书
```
- **政策门**（育儿顾问）：超龄、触红线 → FIX 回炉。
- **理解门**（童读小测者）：遮住文字、像孩子那样只看图，读不通 → 回炉改画面/文字（上限 2 轮）。

## 政策与依据
核心来自《指南》语言领域「阅读与书写准备」——3–4 岁「会看画面，说出图中有谁、发生了什么事」，5–6 岁「专注地阅读图书、据画面线索猜想情节」——这正是「自主翻阅」的官方依据；《指南》亦明确「**机械记忆和强化训练让幼儿过早识字不符合其学习特点**」，故本工程画面为主、文字极简、不做识字训练。内容红线含《未成年人保护法》第五十条与《出版管理条例》未成年人专项条款。详见 `skills/chinese-picture-book/knowledge/`。

## 历史
本工程第一阶段是面向 show-and-tell 表演的**英文**绘本 Skill，已归档于 `legacy/`，其经验证的设计 DNA（图文关系、翻页戏剧、镜头多样、一致性工程、风格库）被改编进新中文 Skill。
