# 定制英文儿童绘本工作流 · 成果总结

> 一句话：把"用 AI 随手生绘本"升级成**一套有标准、可复用、可让客户选风格的儿童绘本生产系统**，
> 专为"3 岁孩子在国外幼儿园 show-and-tell 自己讲"这个场景打造。
> 仓库：https://github.com/Ceshao/Children-s-picture-books

---

## 一、缘起与目标

- **需求**：给在国外生活的姐姐家 3 岁小朋友，定制一本**英文绘本**，拿去幼儿园做 show-and-tell（孩子自己对着图讲、像小演讲）。
- **难点**：此前对绘本的呈现、通用设计、不同年龄的认知规律没有经验；直接用 ChatGPT 生成的版本（V1–V3）家人反馈"不够好"，但说不清问题在哪。
- **目标**：先从产品角度把"什么是好的英文儿童绘本"研究透，抽象成可执行的标准，再固化成一套能反复使用的工作流。

## 二、问题诊断：为什么"直接用 AI 生绘本"不行

把家人模糊的"不够好"拆成了 6 个具体病灶：

| 病灶 | 表现 |
|---|---|
| 文字像旁白 | 文字在复述画面（"他们穿过森林…"），不自然、像念课文 |
| 每页文字太长 | 一页三四句、30+ 词，3 岁记不住、没法复述 |
| 成人式诗意 | "giant leaves whispered in the breeze" 这类拟人隐喻，3 岁听不懂 |
| 每张都是人 | 十页全是主角正面，没有环境页/特写，闷、没节奏 |
| 没有副歌 | 全程新句子，孩子无法预测、无法背诵 |
| 场景跳脱 | 这页森林、下页云端，中间没有过渡 |

## 三、核心思考：从"写小说"到"做表演脚本"

**最关键的认知转向**：AI 直接生绘本时是在"写小说"（连贯、描写、诗意）；而真正优秀的低龄绘本逻辑是**"表演脚本 + 分镜"**——
> 图画讲"发生了什么"，文字只点"一个声音 / 一句对白 / 一个翻页钩子"。让孩子用嘴和身体参与，而不是听一段旁白。

这个判断不是凭空来的，而是**研究了主流英文绘本 + 凯迪克金奖(Medal)/银奖(Honor)作品**后提炼的（并修正了流传的错误，如《市场街最后一站》是银奖非金奖、《晚安月亮》其实未获凯迪克奖、凯迪克是"插画奖"颁给插画家）。

提炼出"**什么是好**"的北极星——研究了 4 本最匹配"3 岁能自己讲"的样板（*We're Going on a Bear Hunt*、*Brown Bear*、*Don't Let the Pigeon Drive the Bus!*、*Goodnight Moon*），归纳成 **6 条能自己感觉的检查**：

> ① 能猜吗（有副歌）② 像玩还是念课文 ③ 能演吗（声音/对白/参与）④ 画面换气了吗（远近明暗在变）⑤ 够小吗（一页一句）⑥ 暖吗（结尾回到安全）

还落实了关键的科学依据：**6 岁以下儿童基本无法理解新颖隐喻**——这科学地解释了"诗意句子为什么不适合 3 岁"。

## 四、交付成果：一套可复用的生产系统

| 成果 | 是什么 |
|---|---|
| **设计标准文档** | 系统性《绘本设计标准与原则》：图文关系 / 年龄认知分级 / 文字与画面工艺 / 凯迪克案例拆解 / 硬性红线 / 反面清单 |
| **北极星标尺** | 一页纸《什么是好·3岁绘本北极星》：4 本样板 + 6 条检查 + 对自己旧版的逐条对照 |
| **风格库** | 8 种可选画风（古典水彩/现代扁平/拼贴/蜡笔/柔和数字/复古限色/柔感3D/水墨），每种配可直接用的生图 prompt，**由客户挑选**（含"风格试片"对比机制） |
| **模板三件套** | 故事圣经（锁角色/世界/风格/副歌）· 分镜表（每页镜头+文字）· 配图 Prompt（真实照片锁脸 + 风格锁定） |
| **演讲套件** | show-and-tell 专用：开场自我介绍、逐页"说什么+做什么"提示卡、**带全班参与的设计**、收尾、Q&A、家长排练计划 |
| **生产检查清单** | 交付前逐项过的红线（含统一画布尺寸、show-and-tell 适配） |
| **可复用 Skill** | 把整套流程封装成 Codex/Claude 可运行的 skill（v1.4.0），**自更新、免重装** |
| **Orion 样例** | 完整跑通的第一本：故事圣经 + before→after 重写 + 逐页配图 prompt + 演讲套件 + 风格试片 |

## 五、关键创新点（展示亮点）

1. **真实照片 → 插画角色**：主角是孩子本人——上传真实照片，先转成插画版"主参考图"，之后每页都附"照片+主参考图"锁住相貌，全书既像本人又风格统一。
2. **"什么是好"被显性化**：不是抽象的"要做好"，而是 6 条能自检、能对照的felt标准 + 4 本可学的样板。
3. **专为演讲设计的"全班参与"**：让 3 岁娃从"紧张背书"变成"带全班一起喊副歌的小主持人"。
4. **风格由客户决定**：8 种画风 + 同一画面切风格的"试片"对比，客户看着实图选。
5. **统一性保障**：全书每页同一画布尺寸；角色/风格锚点每页复述；首尾并排校验防漂移。
6. **工程化**：Skill 自更新（拉取即最新、免重装）、版本管理、GitHub 存档。

## 六、成果验证：Orion 的"旧 → 新"对照（最直观的证明）

| 维度 | 旧版（直接 AI 生成） | 新版（按本系统重做） |
|---|---|---|
| 文字 | 每页 3–4 句旁白 | 每页 1 句、全书 5 次招牌副歌 *"Slow and brave," said Orion.* |
| 语气 | 第三人称描写 | 全对白 + 声音词（crunch×3 / hop×3 / creak×3），可表演 |
| 画面 | 十页人物正面中景 | 13 拍光/色/景别一直"换气"（晨亮→暗→满绿→冷蓝→灰雾→暖光炸开→夜蓝） |
| 结构 | 单向到达就结束 | 原路加速返回（累积回收）+ 暖暖看星星道晚安 |
| 演讲 | 无 | 完整演讲套件：开场自我介绍、带全班接副歌、Q&A、排练计划 |

> 完整对照见 `samples/Orion/`。

## 七、当前状态与下一步

- **现状**：系统、Skill（v1.4.0）、Orion 样例已全部完成并推送到 GitHub；Skill 可在 Codex 拉取即用、自更新。
- **下一步**：
  1. 在 Codex 实跑 Skill，用真实照片 + 真实兴趣生成一本，验证效果；
  2. 用"风格试片"让姐姐一家选定画风；
  3. 出图 → 排版 PDF → 打印成品，交付幼儿园 show-and-tell。
